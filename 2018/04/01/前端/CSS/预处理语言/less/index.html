Less

1、variable

1.1、通用

@link-color: #08aaef;
@hover-color: darken(@link-color, 10%);

a,
a.link {
    color: @link-color;
}

a:hover {
    color: @hover-color;
}

输出为：

a,
a.link {
  color: #08aaef;
}
a:hover {
  color: #0687be;
}

1.2、插值

上面的例子着重于使用变量来控制 CSS 规则中的值，但是它们也可以在其他地方使用，例如选择器名称、属性名称、url 和 @import 语句。

插值的使用形式为  @{variable-name} 拼接其他字符串或者单独使用。

1.2.1、选择器

@btn-prefix: ivu;

.@{btn-prefix}-primary {
    border-radius: 5px;
    border: 1px solid #08aaef;
    color: #08aaef;
}

输出为：

.ivu-primary {
  border-radius: 5px;
  border: 1px solid #08aaef;
  color: #08aaef;
}

1.2.2、属性名

@property-color: color;

.widget {
    @{property-color}: #08aaef;
    background-@{property-color}: #e1e1e1;
}

输出为：

.widget {
  color: #08aaef;
  background-color: #e1e1e1;
}

1.2.3、URLs

@images: '../../assets/images';

.banner {
    background: url('@{images}/home/bg.png') no-repeat;
    background-size: 100% 100%;
}

输出为：

.banner {
  background: url('../../assets/images/home/bg.png') no-repeat;
  background-size: 100% 100%;
}

1.2.4、import

@styles: '../../assets/styles';

@import '@{styles}/variable.less';
@import '@{styles}/mixin.less';

1.3、变量中的变量

在Less中，可以使用另一个变量定义一个变量的名称。

@primary:  green;
@secondary: blue;

.section {
  @color: primary;

  .element {
    color: @@color;
  }
}

输出为：

.section .element {
  color: green;
}

1.4、变量提升

  • Less 中的变量提升与 JavaScript 语言中的变量提升相同
  • Less 中的作用域原则与 JavaScript 语言中的作用域原则相同

2、&

2.1、通用

& 运算符表示嵌套规则中对父选择器的引用。

a {
    color: blue;

    &:hover {
        color: lighten(blue, 10%);
    }
}

输出为：

a {
  color: blue;
}
a:hover {
  color: #3333ff;
}

2.2、插值

.button {
    &-ok {
        color: green;
    }

    &-warning {
        color: yellow;
    }

    &-error {
        color: red;
    }
}

输出为：

.button-ok {
  color: green;
}
.button-warning {
  color: yellow;
}
.button-error {
  color: red;
}

2.3、复合

.button {
    & & {
        color: yellow;
    }

    &>& {
        color: black;
    }

    &+& {
        color: white;
    }

    && {
        color: red;
    }
}

输出为：

.button .button {
  color: yellow;
}
.button > .button {
  color: black;
}
.button + .button {
  color: white;
}
.button.button {
  color: red;
}

2.4、改变顺序

div {
    a & {
        color: red;
    }
}

输出为：

a div {
  color: red;
}

2.5、组合

p, div, a {
    & + & {
        color: red;
    }
}

输出为：

p + p,
p + div,
p + a,
div + p,
div + div,
div + a,
a + p,
a + div,
a + a {
  color: red;
}

2.6、注意

& 表示所有嵌套选择器，而不是最近的父级。如下：

div {
    p {
        a {
            &:hover {
                color: white;
            }
        }
    }
}

输出为：

div p a:hover {
  color: white;
}

而不是：

a:hover {
  color: white;
}

3、extend

extend 是 less 中的一个伪类语法，负责将一个或一些选择器中的声明继承到指定选择器中。

3.1、语法

.b { color:red; }
.a:extend(.b) { background: white; }

意为 a 选择器将继承 b 选择器的规则集，输出为：

.b, .a { color:red; }
.a { background: white; }

该语法的形式有如下几种：

  • 跟在选择器之后：.a:extend(.b)
  • 带空格： .a :extend(.b)
  • 多个串联： .a:extend(.f):extend(.g)
  • 只能跟在最后：因此 pre:hover:extend(div pre):nth-child(odd) 是不允许的
  1. pre:hover:extend(.a):extend(.b) 等同于 pre:hover:extend(.a,.b)
  2. 选择器嵌套情形遵从上面原则

3.2、多重继承

.a:extend(.f){}
.a:extend(.g){}

// 等同于
.a:extend(.f,.g){}

统一输出为：

.f,.a {}
.g,.a {}
.a {}

3.3、精准匹配

默认情况下，less 在为 extend 查找继承选择器时，遵从精准匹配策略。即选择器形式必须完全统一才能匹配，如下列举的情况：

  • .text:extend(.class)  与 .a.class 、.class.a 、.class > .a 、.a > .class、 *.class 不匹配
  • .text:extend(a:hover:visited) 与a:visited:hover 不匹配
  • .text:extend(:nth-child( 1n + 3 )) 与 :nth-child(n+3) 不匹配

但属性选择器的单双引号情况例外：

  • .text:extend([title=identifier]) 与 [title='identifier'] 和 [title="identifier"] 匹配

3.4、all

.a.b.test,
.test.c {
  color: orange;
}
.test {
  &:hover {
    color: green;
  }
}

.replacement:extend(.test all) {}

输出为：

.a.b.test,
.test.c,
.a.b.replacement,
.replacement.c {
  color: orange;
}
.test:hover,
.replacement:hover {
  color: green;
}

3.6、media 作用域

作用域规则可类比 JavaScript 语言中的变量作用域，以 media 花括号为作用域范畴。

3.7、经典场景

3.7.1、减少基类

// CSS
.animal {
    color: white;
    background-color: red;
}

.bear {
    color: black;
}

// HTML
<a classs="animal bear"></a>

可改为：

// CSS
.animal {
    color: white;
    background-color: red;
}

.bear {
  	&:extend(.animal);
    color: black;
}

// HTML
<a classs="bear"></a>

3.7.2、减少重复

.my-inline-block() {
  display: inline-block;
  font-size: 0;
}
.thing1 {
  .my-inline-block;
}
.thing2 {
  .my-inline-block;
}

// 输出为
.thing1 {
  display: inline-block;
  font-size: 0;
}
.thing2 {
  display: inline-block;
  font-size: 0;
}

可改为：

.my-inline-block {
  display: inline-block;
  font-size: 0;
}
.thing1 {
  &:extend(.my-inline-block);
}
.thing2 {
  &:extend(.my-inline-block);
}

// 输出为
.my-inline-block,
.thing1,
.thing2 {
  display: inline-block;
  font-size: 0;
}

4、merge

合并：将多个属性的值合并到单个属性下的逗号或空格分隔的列表中，合并对于诸如背景和变换之类的属性很有用。

// 不合并
.mixin() {
    box-shadow: inset 0 1px 10px #e55;
}

.class {
    .mixin();
    box-shadow: 0 0 20px black;
}

// 输出为
.class {
    box-shadow: inset 0 1px 10px #e55;
    box-shadow: 0 0 20px black;
}

// 合并
.mixin() {
    box-shadow+: inset 0 1px 10px #e55;
}

.class {
    .mixin();
    box-shadow+: 0 0 20px black;
}

// 输出为
.class {
  box-shadow: inset 0 1px 10px #e55, 0 0 20px black;
}

语法：

  • 逗号分隔： + 
  • 空格分隔： +- 

5、mixins

5.1、语法

.class-name, #id-name {
    color: red;
}

.mixin-class {
    .class-name();
}

.mixin-id {
    #id-name();
}

输出为：

.class-name,
#id-name {
  color: red;
}
.mixin-class {
  color: red;
}
.mixin-id {
  color: red;
}

5.2、不输出

.mixin-a {
    color: red;
}

.mixin-b() {
    color: green();
}

.class-a {
    .mixin-a();
    .mixin-b();
}

输出为：

.mixin-a {
  color: red;
}
.class-a {
  color: red;
  color: green;
}

5.3、Selectors in Mixins

Mixins 不仅可以包含属性，还可以包含其他的选择器。

.mixin-hover() {
    color: red;

    &:hover {
        background-color: white;
    }
}

button {
    .mixin-hover();
}

输出为：

button {
  color: red;
}
button:hover {
  background-color: white;
}

5.4、Namespace

  • 减少与其他库混合使用时的冲突
  • 一种组织 mixins 的方法
#outer() {
  .inner-a {
    color: red;
  }
  .inner-b {
    color: green;
  }
}

.c {
  #outer > .inner-a();
}

5.5、守卫

#namespace when (@mode = huge) {
  .mixin() { /* */ }
}

#namespace {
  .mixin() when (@mode = huge) { /* */ }
}

5.6、important

.foo(@bg: #e55, @color: #4e5) {
    background-color: @bg;
    color: @color;
}

.unimportant {
    .foo();
}

.important {
    .foo() !important;
}

输出为：

.unimportant {
  background-color: #e55;
  color: #4e5;
}
.important {
  background-color: #e55 !important;
  color: #4e5 !important;
}

6、import

  • @import 语句可以在文档的任何位置（区别于 css 中的只能在头部）
  • 根据文件扩展名的不同， import 语句的处理方式可能会有所不同
    • @import “foo”; // foo.less is imported
    • @import “foo.less”; // foo.less is imported
    • @import “foo.php”; // foo.php imported as a Less file
    • @import “foo.css”; // statement left in place, as-is

6.1、选项

Syntax: @import (keyword) "filename";

  • reference: use a Less file but do not output it
  • inline: include the source file in the output but do not process it
  • less: treat the file as a Less file, no matter what the file extension
  • css: treat the file as a CSS file, no matter what the file extension
  • once: only include the file once (this is default behavior)
  • multiple: include the file multiple times
  • optional: continue compiling when file is not found
